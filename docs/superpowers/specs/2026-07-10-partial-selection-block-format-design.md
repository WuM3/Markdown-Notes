# 部分选区的引用与代码块转换

## 目标

当用户的文本选区从一个段落（或其他可转换文本块）中间开始，并在另一个文本块中间结束时，“引用”和“代码块”工具应处理全部被触及的文本块，而不是仅处理选中的字符片段。

这里的“行”是编辑器中独立的文本块，例如截图中的三段独立文字。浏览器因窗口宽度造成的同一段自动换行不作为新的行，保证桌面端与移动端的结果一致。

## 行为

- 将选区扩展到起始文本块的开头和结束文本块的结尾。
- 点击“引用”时，仅把扩展范围内的完整文本块转换或包裹为引用；相邻、未触及的文本块保持不变。
- 点击“代码块”时，读取扩展范围内完整文本块的纯文本内容，并替换为一个代码块；代码语言继续沿用现有自动推断逻辑。
- 空选区保持现有行为：引用切换当前块，代码块转换当前块。
- 只包含空白字符的非空选区不改变内容。

## 实现边界

在 `toolbar-commands.ts` 中集中处理选区扩展，供引用和代码块工具使用。扩展逻辑基于 ProseMirror 文档节点边界，不依赖浏览器布局或像素坐标。

## 验证

为命令层增加测试：构造从首个文本块中间到最后一个文本块中间的选区，断言引用仅作用于被触及的完整块，代码块包含被触及块的全部文本。现有全量单元测试、类型检查与构建必须继续通过。
