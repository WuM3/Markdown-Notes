# 响应式文档目录设计

## 目标

让桌面端文档目录根据编辑区的可用宽度缩放，而不是依赖显示器视口宽度；长标题在所有视口保持单行并以省略号截断。桌面端悬停标题时显示浏览器原生完整标题提示，移动端与平板抽屉只保留省略号。

## 方案

- 将桌面 `.document-canvas` 的目录列从基于 `vw` 的宽度改为相对网格容器的百分比，并使用最小/最大宽度限制；正文列保持 `minmax(0, 860px)`，所以侧栏、窗口和显示器尺寸变化时不会产生横向溢出。
- 将 `.document-outline-link` 从 `text-overflow: clip` 改为 `ellipsis`；现有 `min-width: 0`、`overflow: hidden` 与 `white-space: nowrap` 继续确保按钮可收缩。
- 给标题按钮设置 `title={node.title}`。桌面浏览器显示原生悬停完整标题；触摸设备不会依赖此提示。

## 验证

- 组件测试断言标题按钮具有完整的 `title` 属性，且可访问名称和导航行为保持不变。
- 更新目录 E2E 检查：长标题使用省略号、桌面端目录和页面没有横向溢出，并在桌面/平板/移动目录中保留单行样式。
- 按 UI 改动分级运行受影响 Vitest、受影响 Playwright、类型检查和生产构建。
