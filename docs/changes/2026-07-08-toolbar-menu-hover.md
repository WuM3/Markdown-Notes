# 2026-07-08 工具栏菜单悬浮修复

## 修改内容

- 修复选中文字后工具栏的颜色菜单和格式菜单在鼠标移向面板时立即消失的问题。
- 菜单离开按钮区域后增加 160ms 的短暂关闭延迟，鼠标进入菜单面板会取消关闭。
- 点击菜单外部仍会立即关闭，选择颜色或格式后也会正常关闭。

## 验证

- 增加回归测试覆盖从按钮穿过间隙进入菜单面板的场景。
- 增加回归测试覆盖离开按钮和菜单后延迟关闭的场景。

## 颜色菜单修复

- 修复字体颜色和背景颜色点击后没有作用到原选中文字的问题。
- 打开颜色菜单时保存编辑器当前选区，点击色块时用该选区应用样式，避免菜单按钮夺焦导致选区丢失。
- 颜色菜单会根据当前选中文字的字体颜色和背景颜色高亮对应色块。

## 颜色样式与跨行选择数据安全修复

- 修复字体颜色和背景颜色保存链路：样式标记不再直接吞掉 Markdown 文本节点，而是按普通 mark 序列化后交给 remark handler 输出 `<span style="...">`。
- 修复跨行或跨段落改色后可能导致文字消失、页面卡死、刷新后内容异常的问题，重点保护正文内容不因样式失败而丢失。
- 保存后的 `<span style="...">` 会在重新加载 Markdown 时恢复成编辑器内的颜色标记，并保留内部加粗等嵌套 Markdown 内容。
- 对非法颜色样式做防护：已知颜色属性如果包含危险或异常值，会被整体拒绝，不进入编辑器 mark。
- 增加选区兜底：工具栏保存的选区失效或越界时，不会把样式应用到错误范围，空选区只影响后续输入。

## 新增验证

- 增加 Markdown 序列化测试，覆盖多段落颜色文本不会被保存时吞掉。
- 增加 Markdown 反序列化测试，覆盖合法 style span 恢复、未闭合 span 原样保留。
- 增加样式解析安全测试，覆盖 url、异常字符和超长值。
- 增加命令层测试，覆盖失效选区、恢复默认、空选区后续输入和工具栏夺焦后的保留选区。

## 稳定选区捕获补充

- 工具栏打开颜色菜单时，如果 ProseMirror 当前选区已经被工具栏焦点影响为空，会尝试从浏览器 DOM selection 反推编辑器选区，避免颜色只作用到后续输入。
- 颜色菜单读取当前样式失败时仍会正常打开，避免临时选区异常导致菜单只剩 tooltip、不显示面板。
- 新增 DOM selection 兜底测试和菜单异常兜底测试，覆盖选区在编辑器内、编辑器外和当前样式读取异常的边界情况。

## 颜色 Mark 注册补齐

- 修复上一次因写入权限不足未落到生产代码的问题，`MarkdownEditor` 现在会在 `textStyleSchema` 之前注册 `textStyleAttr`。
- 该注册顺序与 Milkdown 内置 mark 的 attr/schema 模式保持一致，避免颜色 mark 命令执行后无法正常渲染到编辑器 DOM。
- 新增编辑器注册顺序测试，防止后续重构时漏掉 `textStyleAttr` 或把它放到 schema 之后。

## 颜色菜单双触发修复

- 修复色块同时响应 `pointerdown` 和同一次鼠标点击生成的 `click`，导致一次操作执行两次颜色应用的问题。
- 保留 `pointerdown` 以避免工具栏夺焦丢失选区，同时忽略重复的鼠标 `click`；键盘触发的 `click` 仍然可用。
- 增加回归测试覆盖鼠标点击只应用一次颜色，以及键盘点击仍能应用颜色。
- 增加桌面端端到端测试，覆盖选中文字后应用字体颜色，实时编辑器内文字仍保持可见。
