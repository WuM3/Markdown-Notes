# 文档目录布局与显示优化

日期：2026-06-27

## 本次改动

- 将文档目录从编辑器右侧移动到编辑器左侧，更接近飞书文档目录的阅读习惯。
- 桌面端文档目录支持整体收起和展开。
- 收起后只保留一个窄栏按钮，点击“展开文档目录”即可恢复完整目录。
- 有子标题的目录项仍支持展开和收起子标题。
- 子标题展开/收起的小三角默认隐藏，鼠标悬浮到标题行或键盘聚焦时才显示。
- 手机端继续使用“文档目录”抽屉，触屏场景下小三角保持可见，避免无法操作。
- 文档目录标题改为完整显示，不再使用省略号截断。
- 长标题保持单行显示，目录区域可以横向滚动查看完整标题。

## 追加调整：目录并入编辑画布

- 将桌面端文档目录从独立侧栏结构改为编辑滚动画布内部的一部分。
- 文档目录和正文共享同一个白色页面背景和滚动上下文，减少与正文之间的割裂感。
- 桌面端去掉“文档目录”标题栏，只保留类似飞书的收起按钮。
- 收起文档目录时只渲染窄按钮列，不再让目录文字参与宽度压缩动画。
- 展开文档目录时直接恢复完整目录内容，避免展开过程中出现文字闪烁或挤压。
- 手机端仍保留抽屉式文档目录，不影响窄屏编辑空间。

## 追加调整：去除左侧空白和横向滚动

- 取消桌面端文档画布的居中宽度限制，让文档目录从编辑区域左侧自然开始显示。
- 将桌面端文档目录列加宽，常见较长标题可以保持单行完整显示。
- 删除文档目录底部的横向滚动条，目录标题不再通过横向拖动查看。
- 保留标题单行显示，不再主动换行。
- 文档目录整体收起和展开时增加 180ms 左右方向过渡，减少突然跳变感。
- 收起状态只保留窄按钮列，展开状态恢复完整目录列。

## 追加调整：滚动与目录收起

- 修复桌面端正文鼠标滚轮无法正常滚动的问题。
- 编辑区域现在明确由正文滚动层负责纵向滚动，避免内容撑开布局后被外层裁掉。
- 鼠标停在文档目录上滚动时，也会带动正文滚动。
- 文档目录列略微缩窄，让正文整体稍微左移到更接近居中的位置。
- 收起文档目录时不再改变正文列位置，正文不会突然跳到最左侧。
- 左侧文件夹目录新增收起按钮。
- 文件夹目录收起后保留一个窄列展开按钮，点击即可恢复目录树。

## 追加调整：目录固定与标题字重

- 文档目录在正文滚动时保持固定，不再只停在页面顶部后随正文滚出视野。
- Markdown 正文中的 H1-H5 标题强制保持加粗显示，即使对标题文字使用加粗按钮也不会取消标题字重。
- 文档目录中的顶层标题项强制加粗，便于区分一级结构。
- 左侧文件夹目录展开时使用固定宽度内容层，展开动画只裁剪外层宽度，避免文字在过渡过程中被挤压显示。

## 保持不变

- 文档目录仍然从当前 Markdown 的 H1-H5 自动生成。
- 目录展开/收起状态只保存在前端界面中，不写入 Markdown 文件。
- 点击目录项仍会定位到对应标题。
- 移动端仍通过顶部“文档目录”按钮打开抽屉。
