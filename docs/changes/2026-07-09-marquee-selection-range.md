# 2026-07-09 框选选区范围修复

## 修复问题

- 修复从正文右侧空白处拖出蓝色框选区域后，松开鼠标时偶发选中整段的问题。
- 之前蓝色高亮按“被覆盖的行”显示，但真实 ProseMirror 选区按“命中的整个顶层块”生成，导致只框到段落中间几行时也可能把整段选中。

## 修改内容

- 框选松开后的真实选区改为基于命中行的坐标计算起止位置。
- 普通段落和标题只选中蓝框覆盖到的起始行与结束行之间的文本。
- 图片单独被框中时仍按图片节点选中，便于复制和后续操作。
- 代码块参与混合框选时按完整代码块边界选中，避免把代码块内部状态拆坏。
- 框选起点不再从代码块、图片块和 CodeMirror 编辑区域触发，减少和内部编辑/选择逻辑冲突。

## 测试覆盖

- 单段落多行换行时，只框中中间几行不会选中整段。
- 混合命中段落、图片、代码块和标题时，首尾文本块按行裁剪，中间图片/代码块按块选中。
- 单独框中图片时生成图片节点选区。
- 从代码块开始或结束的框选使用代码块完整边界。
- 新增桌面端 Playwright 回归：先让编辑器失焦，再从正文左侧空白向右拖框覆盖长段落中间一行，确认选区可见且不会选中整段。

## 失焦后选区可见性

- 修复从正文两侧空白处开始框选时，编辑器处于失焦状态，松开鼠标后没有可见选区的问题。
- 框选完成时先恢复编辑器焦点，再提交 ProseMirror 选区事务，使浏览器原生高亮与浮动格式工具栏由同一次状态更新正常显示。
