# 2026-06-28 图片显示与选择交互

## 新增功能

- 图片块选中后显示蓝色边框和四个角的缩放控制点。
- 按住图片四角拖动时，会按比例调整图片显示高度，并继续保存到 Milkdown 原有的 Markdown 图片比例中。
- 在编辑器空白区域按住鼠标拖动时，会显示蓝色框选区域；框到单张图片时可快速选中图片，框到多个块时会尽量生成连续选区。
- 选中图片后复制时，优先向系统剪贴板写入真实图片数据；同时写入 HTML 图片和纯文本链接作为浏览器兼容兜底。
- 空白处拖动框选升级为飞书式实时行级高亮：拖动时保留右侧蓝色框，同时对被框到的文字行铺浅蓝色选区，松开后转成真实编辑器选区。

## 修改功能

- 修复宽图或缩放后的图片被裁剪、显示不全的问题。
- 图片显示改为保持原始宽高比并适配正文宽度，不再使用裁剪填充效果。
- 隐藏 Milkdown 默认的底部图片缩放条，统一使用四角缩放控制点。
- 图片缩放改为按宽度比例显示，高度自动计算，避免 `object-fit: contain` 在固定高盒子里产生上下空白。
- 图片最大缩放比例限制为正文可容纳的最大宽度，拖到最大后不会继续放大。
- 四角拖拽改为沿对角线平滑计算比例，减少拖动时突然跳变的问题。
- 图片最大显示宽度改为笔记正文可用宽度，不再受图片原始宽度限制，较小的图片也可以放大到正文横向最宽处。
- 取消图片四角缩放的启动阈值，按住角点后从第一段有效拖动开始连续缩放，减少刚开始拖动时的停顿感。
- 修复空白框选触发范围过窄的问题：现在从段落、标题等文本块右侧空白处拖动也会出现蓝色框选区域，并避免被 ProseMirror 默认拖拽选区抢走。
- 框选时会根据文本真实换行区域计算高亮，不再只是显示一个孤立矩形框，拖动反馈更接近飞书。

## 说明

- 图片大小仍保存在 Markdown 图片语法的 alt 比例中，例如 `![1.52](.assets/...)`，不额外引入私有数据库或新格式。
- 复制真实图片依赖浏览器剪贴板能力；如果浏览器或访问环境不允许写入图片 Blob，会自动保留 HTML/文本兜底。
